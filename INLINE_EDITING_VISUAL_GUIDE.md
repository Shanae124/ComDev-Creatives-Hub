```
Step 1: Navigate
┌──────────────┐
│ Go to        │
│ /pages-      │

        ↓

┌──────────────────────────────────────┐

┌──────────────────────────────────────┐
│ Click "Enable Edit Mode" button      │
│ (turns blue when active)             │
└──────────────────────────────────────┘
        ↓

Step 4: Click Element to Edit
┌──────────────────────────────────────┐
│ Hover over text in preview           │
│ → Blue dashed outline appears        │
│ Click to select                      │
└──────────────────────────────────────┘
        ↓

Step 5: Edit Text
┌──────────────────────────────────────┐
│ Type new text in textarea below      │
│ Edit multiple times if needed        │
└──────────────────────────────────────┘
        ↓

Step 6: Update
┌──────────────────────────────────────┐
│ Click "Update Text" button           │
│ Preview updates instantly            │
│ HTML code syncs automatically        │
└──────────────────────────────────────┘
        ↓

Step 7: Add More Content (Optional)
┌──────────────────────────────────────┐
│ Click "+ Paragraph"  → Add paragraph │
│ Click "+ Heading"    → Add heading   │
│ Click "+ Section"    → Add container │
└──────────────────────────────────────┘
        ↓

Step 8: Save
┌──────────────────────────────────────┐
│ Click "Save Page" button             │
│ Page saved to database               │
│ Students see updated content         │
└──────────────────────────────────────┘
```

## 🖱️ Mouse Actions

### WYSIWYG Editor

```
HOVER:
Mouse over element in preview
        ↓
Element border changes from none to:
    2px dashed #00bcd4 (cyan)
    outlineOffset: 2px
        ↓
Visual feedback: "This element can be edited"

CLICK:
Click on highlighted element
        ↓
Element selected (state updates)
selectedElement = HTMLElement
editingText = element.innerText
        ↓
Editor panel appears below with:
- Label: "Editing: [TAG NAME]"
- Textarea with current text
- [Update Text] button
- [+ Paragraph] button
- [+ Heading] button

CLICK "Update Text":
        ↓
selectedElement.innerText = editingText
getNewHTML from iframe
setHtmlCode(newHTML)
        ↓
HTML code synchronized
Preview updated
```

### Content Editor

```
HOVER on section title:
        ↓
Background changes to light hover color
Cursor becomes pointer
        ↓
Visual feedback: "Click to edit"

CLICK on section title:
        ↓
Title text replaced with <input>
Input focused (autoFocus={true})
        ↓
User types new title
        ↓
User presses Enter or clicks away
        ↓
Input replaced with new title text
State updated

CLICK on lesson content:
        ↓
Content replaced with <textarea>
HTML editor appears
        ↓
User edits HTML
        ↓
User clicks "Update Text"
        ↓
Textarea replaced with new content
State updated
```

## ⌨️ Keyboard Experience

### WYSIWYG Editor

```
Tab Key:
Current Element → Focus moves to next interactive element

Shift+Tab:
Current Element → Focus moves to previous element

Enter:
In some text fields → Confirms entry

Escape:
In edit mode → Cancel editing (optional)
```

### Content Editor

```
Click on title → Input focused

Type → Title text changes

Enter → Title saved, input closes

Tab → Move to next field

Delete Lesson → Requires button click (for safety)
```

## 🎯 Comparison: Old vs New Workflow

### Old Workflow: Edit Course Page

```
1. Download course package (2-3 min)
   [Choose format] → [Download] → [Extract]

2. Open in editor (1 min)
   [Locate file] → [Open in editor]

3. Find and edit content (5 min)
   [Search for text] → [Find element] → [Edit]

4. Re-import to system (2-3 min)
   [Package] → [Upload] → [Verify]

5. Test in student view (2-3 min)
   [Publish] → [Switch roles] → [View]

TOTAL: 12-18 MINUTES per change
COMPLEXITY: High
ERROR RATE: Medium (re-import issues)
```

### New Workflow: Edit Course Page

```
1. Click "Edit Content" (5 seconds)
   [Click button]

2. Click element to edit (3 seconds)
   [Click in preview]

3. Type changes (2-5 minutes)
   [Edit in textarea]

4. Save (2 seconds)
   [Click Save]

5. View live (1 second)
   [See preview update]

TOTAL: 3-6 MINUTES per change
COMPLEXITY: Very Low
ERROR RATE: None (live preview prevents errors)

IMPROVEMENT: 50-75% faster, much simpler
```
